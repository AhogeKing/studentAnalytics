# 2026-06-25

## 前端范围

当前前端只启用后端已经实现的学生相关能力，暂不展示仪表盘、预测分析、批量导入、新增学生等未接通后端的入口。

已启用页面：

- 登录 / 注册页：`/login`
- 学生概览页：`/students`
- 学生详情页：`/students/:studentNo`
- 学生概览编辑弹窗：由概览页和详情页打开

## 技术栈

- Vue 3 + TypeScript
- Vue Router
- Pinia
- Element Plus
- ECharts 依赖保留，但当前学生概览页面不启用图表型仪表盘
- Axios 请求后端接口

## 视觉方向

整体按现代 SaaS 后台管理界面处理，优先保证信息密度、可扫描性和工作流效率。

设计要点：

- 主色使用青色系和少量绿色点缀，避免过重的深色仪表盘视觉。
- 页面结构保持轻量：顶部导航、查询区、内容区、详情信息区。
- 卡片圆角控制在 8px，表格和表单控件保持稳定尺寸。
- 登录页标题为“学生学业信息管理系统”，与登录框居中对齐，标题保持单行显示。
- 登录框内提供“登录 / 注册”模式切换，注册成功后回到登录模式。

## 接口映射

认证：

- `POST /login`：登录，保存 token 和用户信息
- `POST /register`：注册教师账号，前端提交 `username`、`password`、`realName`
- `POST /logout`：退出登录
- `GET /me`：刷新当前用户信息

学生：

- `GET /students/page`：学生概览查询
- `GET /students/filter-options`：筛选选项
- `GET /students/detail/{studentNo}`：学生详情
- `PUT /students/overview/{studentNo}`：编辑学生概览字段
- `DELETE /students/overview/{studentNo}`：删除学生概览记录

## 页面规则

### 登录 / 注册页

- 登录成功后跳转到 `/students`。
- 注册不单独开路由，直接在登录框内切换模式。
- 注册角色不在前端暴露，由后端默认处理为 `TEACHER`。
- 注册成功后清空确认密码并回到登录模式。

### 学生概览页

- 默认进入高一 1 班，即查询参数 `class_name=1-1`。
- 保留班级轮播功能：按班级列表顺序每 5 秒切换下一班。
- 搜索模式和筛选模式互斥：
  - 搜索模式只提交 `keyword` 和排序参数。
  - 筛选模式提交班级、年级、性别、GPA、成绩等级和排序参数。
- 后端返回完整结果后，前端按 20 条一页分页展示。
- 表格支持后端允许的排序字段：`studentNo`、`name`、`age`、`gpa`、`gradeClass`。
- 不显示“新增学生”按钮，因为后端目前没有新增学生接口。

### 学生详情页

详情页分为四块：

- 基础信息
- 学业表现
- 支持状态
- 活动画像

当 `performance_available=false` 时，只展示基础信息，其余区域显示空状态。

### 编辑弹窗

编辑弹窗只暴露当前后端允许且设计确认的字段：

- 姓名
- 年龄
- 性别
- 班级
- GPA

不暴露也不提交：

- `grade_level`
- `grade_class`

提交时只发送发生变化的字段。没有变化时不调用更新接口。

## 权限显示

前端按用户角色做基础显示控制：

- `admin`：展示编辑和删除按钮
- 非 `admin`：只展示查看详情能力

最终权限仍以后端校验为准。

## 分析图表页面

- 页面根据 `doc/2026-06-25-analysis-charts-design.md` 的前端设计实现五张图：
  - GPA 区间人数气泡图；
  - GradeClass 成绩等级柱状图；
  - 缺勤次数与 GPA 散点图；
  - 每周学习时长与 GPA 散点图；
  - 学习投入、缺勤与 GPA 联合分布。
- 前端 API 封装为 `frontend/src/api/analytics.ts`，当前实际后端路径为 `/analytics/*`。

- 散点图支持点击学生点跳转到学生详情页。
- 图表卡片统一处理 loading、空数据、错误重试和响应式 resize。
- 当前后端分析 VO 未显式使用 `@JsonProperty`，前端类型和归一化函数同时兼容 camelCase 与 snake_case 字段。

## 分析图表颜色聚焦

- 新增散点图公共控制栏，区分三个前端状态：
  - 着色依据：成绩等级 / GPA 区间；
  - 聚焦学生：全部 / 当前着色依据下的各个分组；
  - 显示方式：弱化其他学生 / 仅显示选中学生。
- 颜色聚焦属于前端表现层，不向后端追加 `grade_class` 或 `gpa_bucket` 过滤参数；后端年级、班级筛选仍负责决定本次分析包含哪些学生。
- 三张散点图共用同一套 `colorMode`、`focusedGroup`、`displayMode` 状态，点击控制栏后同步更新。
- 散点图改为按分组生成多个 ECharts `series`，聚焦时通过透明度和层级突出目标组。
- 当前后端如果未返回 `gpaBucket/gpa_bucket`，前端按 0.5 GPA 间隔本地计算区间；后端补齐字段后可无缝使用后端值。
- 单击散点聚焦该学生所属分组，双击散点进入学生详情，避免单击同时承担两个交互动作。
- 点击散点图顶部图例图标时，前端按图例分组聚焦学生；再次点击当前分组时，在“弱化其他学生”和“仅显示选中学生”之间切换。
- GPA 区间 `[3.0, 3.5)` 使用天蓝色，和 `[3.5, 4.0]` 深蓝色拉开可视差异。
- 聚焦分组和切换显示方式时，图表组件只增量更新 `legend` 与 `series`，不替换 x/y 坐标轴配置。
- “仅显示选中学生”保留非目标分组数据但设置为透明和静默，避免 ECharts 因数据范围变化重新计算坐标轴。
- 三张散点图的 x/y 轴范围基于完整 `performance-points` 数据计算，不随当前聚焦分组缩放。
- 散点图图例图标尺寸从 9px 增加到 12px，降低点击落空概率。
- 散点图顶部图例改为 HTML 覆盖层按钮，视觉保持小圆点图例，但点击不再依赖 ECharts canvas 命中，降低图标点击无响应概率。
- 成绩等级颜色改为模仿 GPA 区间的低到高语义：风险红色、较差橙色、中等绿色、良好天蓝色、优秀深蓝色。
