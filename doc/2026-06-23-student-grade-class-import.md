# 2026-06-23 学生年级与班级导入规则

## 背景

为了支持按年级、班级分页或筛选查询，`student` 表新增两个字段：

- `grade_level`：年级，取值为 `1`、`2`、`3`，分别表示高一、高二、高三。
- `class_name`：班级，格式为 `年级-班级`，例如 `1-3` 表示高一 3 班，`3-12` 表示高三 12 班。

`student_performance.grade_class` 仍表示成绩等级，编码规则为 `0=A, 1=B, 2=C, 3=D, 4=F`，不要与学生年级混用。

## 年级分配规则

导入 CSV 时根据年龄分配年级：

- 15 岁固定分配到高一，即 `grade_level = 1`。
- 18 岁固定分配到高三，即 `grade_level = 3`。
- 16 岁在高一和高二之间均衡随机分配。
- 17 岁在高二和高三之间均衡随机分配。

分配使用固定随机种子，保证重复导入时结果可复现。

## 班级数量规则

当前导入采用以下班级数量：

| 年级 | 班级范围 | 班级数量 |
|---|---|---:|
| 高一 | `1-1` 到 `1-17` | 17 |
| 高二 | `2-1` 到 `2-12` | 12 |
| 高三 | `3-1` 到 `3-17` | 17 |

班级内人数按年级内均衡随机分配，优先把学生分配到当前人数较少的班级。

## 成绩等级处理

CSV 中最后一批数据存在 GPA 与 `GradeClass` 不一致的问题。导入时以 GPA 为准重新计算 `student_performance.grade_class`：

| GPA 范围 | grade_class |
|---|---:|
| `3.5 <= GPA <= 4.0` | 0 |
| `3.0 <= GPA < 3.5` | 1 |
| `2.5 <= GPA < 3.0` | 2 |
| `2.0 <= GPA < 2.5` | 3 |
| `GPA < 2.0` | 4 |

如果 CSV 原始 `GradeClass` 与 GPA 计算结果不一致，导入后的 `grade_class` 使用 GPA 计算值，同时保留 `data_quality_status = 1` 与 `quality_issue` 作为数据质量提示。

## 导入前清理规则

每次执行 `python/importer/student_data.py` 导入前，会在同一事务中先清理旧数据：

1. 删除 `student_performance` 数据。
2. 删除 `student` 数据。
3. 重置 `student_performance` 自增主键。
4. 重置 `student` 自增主键。
5. 重新插入 CSV 数据。
